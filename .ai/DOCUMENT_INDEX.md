# Documentation Index

The current knowledge base lives in `documentation/`. Use the following sources by purpose.

| Need | Primary source |
| --- | --- |
| Project identity, vision, philosophy | `documentation/PROJECT_BIBLE.md`, `documentation/00_Project/` |
| System and entity boundaries | `documentation/architecture/` |
| Exact planned behavior and acceptance criteria | `documentation/specs/` |
| Significant accepted decisions | `documentation/history/ADR/` and `documentation/DESIGN_DECISIONS.md` |
| Engineering and documentation standards | `documentation/development/` |
| Roadmap and phase planning | `documentation/ROADMAP.md` and `documentation/roadmap/` |
| Historical context and implementation record | `documentation/history/` |
| User- and contributor-facing guides | `documentation/wiki/`, `README.md`, `CONTRIBUTING.md` |

`documentation/PROJECT_INDEX.md` describes the intended hierarchy. When its paths differ from the repository's current folder names, use the files and directories that actually exist in `documentation/` and report the discrepancy rather than silently moving content.
