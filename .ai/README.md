# HiveMind AI Contributor Guide

This directory provides AI-agnostic onboarding guidance for contributors to the HiveMind module of the Living Systems Project (LSP).

Read these files before proposing, documenting, or implementing a change:

1. [PROJECT.md](PROJECT.md)
2. [DOCUMENT_INDEX.md](DOCUMENT_INDEX.md)
3. [DOCUMENT_RULES.md](DOCUMENT_RULES.md)
4. [AI_GUIDELINES.md](AI_GUIDELINES.md)
5. [STYLE_GUIDE.md](STYLE_GUIDE.md)
6. [WORKFLOW.md](WORKFLOW.md)
7. [TASKS.md](TASKS.md)

This directory does not replace the existing documentation in `documentation/`. It is a navigation and contribution layer for humans and AI tools.

## Authority

Project direction comes from the maintainer. Accepted architecture is recorded in the project documentation and ADR history. Do not introduce new gameplay systems or materially change architecture without explicit approval.
