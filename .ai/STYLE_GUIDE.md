# Documentation and Change Style

The canonical documentation style is defined in `documentation/development/DOCUMENTATION_STYLE_GUIDE.md`. Follow it for all new or updated project documentation.

## Local rules

- Use Markdown and clear ATX headings.
- Write for future human and AI contributors: explain purpose, rationale, behavior, and relationships when relevant.
- Use concise prose, lists, tables, and Mermaid diagrams only when they improve understanding.
- Avoid duplicate sources of truth; link to the authoritative document instead.
- Preserve established terminology, capitalization, and file naming within the relevant documentation area.
- Update metadata and version history only when the owning document uses them and an actual change warrants it.
