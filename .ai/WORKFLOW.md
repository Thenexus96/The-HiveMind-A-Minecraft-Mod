# Contribution Workflow

1. Inspect repository status and relevant guidance before editing.
2. Identify the authoritative documentation source for the task using `DOCUMENT_INDEX.md`.
3. Summarize the intended change and flag conflicts or missing decisions.
4. Make the smallest implementation or documentation change that fulfills the approved task.
5. Verify links, formatting, and relevant build or test results.
6. Report files changed, verification performed, and unresolved follow-up work.

For architecture changes, record the decision through the project's existing decision/ADR process before treating the implementation as canonical.
